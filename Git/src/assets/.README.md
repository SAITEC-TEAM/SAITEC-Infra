# 项目使用说明

## 初始化时做了什么
- 生成 `pyproject.toml`
- 复制 `.gitignore`、`.gitattributes`、`.pre-commit-config.yaml`、`.README.md`
- 创建 `.vscode`、`.cursor`、`.trae`
- 写入 `.vscode/settings.json`
- 安装 Git hooks
- 如果有 `requirements.txt`，导入到 `pyproject.toml`
- 如果启用了 AI 模板，再额外复制对应协作文档

## 怎么开发
- 先准备 Python 环境，并安装项目依赖
- `taskipy` 命令写在 `pyproject.toml` 的 `[tool.taskipy.tasks]`
- 当前常用命令：
  - `python -m taskipy test`
  - `python -m taskipy check`
  - `task test`
  - `task check`
- 如果新增了任务，按同样格式执行：`python -m taskipy <任务名>` 或 `task <任务名>`

## 怎么提交
- 在功能分支开发，不要直接在 `main` 上改
- 提交前先运行 `check` 和 `test`
- `git add`、`git commit`、`git push`
- 推送后走 Pull Request 合并，不直接推送到 `main`

## 需要遵守的规范
- 分支名必须是 `feat/<topic>`、`fix/<topic>` 或 `new/<topic>`
- `pre-commit` 会在提交前检查分支名，并自动执行格式化
- `pre-push` 会阻止直接推送到远程 `main`
- 变更要尽量小、清晰，并同步更新必要文档
